# cloudnative-pg

CloudNativePG operator + Barman Cloud Plugin manifests, deployed via Kustomize overlays. The operator manages the full lifecycle of PostgreSQL clusters as native Kubernetes resources — replacing per-cluster RDS instances over time as part of the broader cloud-portability move.

## Layout

```
cloudnative-pg/
  vendir.yml                        # Pins external manifest artifacts
  vendir.lock.yml                   # Lockfile, regenerated by `vendir sync`
  base/
    namespace.yaml                  # cnpg-system namespace
    kustomization.yaml              # helmCharts: [cloudnative-pg], values + resources, patches
    vendor/                         # vendir-managed (committed)
      plugin-barman-cloud/
        manifest.yaml               # synced from upstream releases
  overlays/
    oke/                            # OKE deployment, no patches today
      kustomization.yaml
```

The operator is rendered from the upstream Helm chart at `kustomize build` time via `helmCharts:`. The Barman Cloud Plugin is a static manifest pulled from upstream GitHub releases via `vendir`. Both upgrade workflows are bump-the-version-and-resync — no `helm template > file.yaml` step.

## CRDs Provided

- `Cluster` — PostgreSQL cluster definition
- `Backup` — On-demand backup
- `ScheduledBackup` — Scheduled backups
- `Pooler` — PgBouncer connection pooling
- `ObjectStore` — Backup destination configuration (from Barman Cloud Plugin)

## Verifying a render

```bash
kubectl kustomize --enable-helm overlays/oke
```

The output should match the live cluster's resources after sync. ArgoCD passes `--enable-helm` to its kustomize builder, so this matches what ArgoCD will apply.

## Operator Upgrade Procedure

1. Check release notes: https://cloudnative-pg.io/documentation/current/release_notes/
2. Bump `helmCharts[0].version` in `base/kustomization.yaml` to the new chart version.
3. (If chart values changed) update `valuesInline:` to match.
4. Verify locally: `kubectl kustomize --enable-helm overlays/oke`.
5. Commit and push. Open a PR.
6. After merge, bump `targetRevision` on the `cloudnative-pg` ArgoCD Application in `internal-applications-kub`.

## Barman Cloud Plugin Upgrade Procedure

1. Check release notes: https://github.com/cloudnative-pg/plugin-barman-cloud/releases
2. Bump `tag` in `vendir.yml` to the new release.
3. Run `vendir sync` from the repo root. This rewrites `base/vendor/plugin-barman-cloud/manifest.yaml` and `vendir.lock.yml`.
4. Verify locally: `kubectl kustomize --enable-helm overlays/oke`.
5. Commit (including the updated `vendor/` content) and push. Open a PR.
6. After merge, bump the Application `targetRevision`.

## Resource sizing

The kustomization sets `resources.requests` and `resources.limits` on both Deployments based on observed peak memory in the existing OKE deployment over a 30-day window:

| Container | Observed peak | Request | Limit |
|---|---|---|---|
| `manager` (operator) | 74 MiB | 50m / 96Mi | 500m / 256Mi |
| `barman-cloud` | 32 MiB | 25m / 64Mi | 250m / 192Mi |

Headroom is generous on memory (3.5×) because Prometheus rate sampling doesn't fully capture the spikes during cluster-reconcile bursts. Revisit once we have CNPG-managed Clusters producing real load.
